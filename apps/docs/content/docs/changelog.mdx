## v0.1.0 — initial public release

First public release. The seven `@gemmapod/*` packages and two Docker
images publish together.

### Packages

| Package              | Notes                                                                                       |
| -------------------- | ------------------------------------------------------------------------------------------- |
| `@gemmapod/core`     | Rust → WASM. Two subpath exports (`/web`, `/node`). Sigstore provenance.                    |
| `@gemmapod/dartc`    | DARTC v0.2 envelope types, canonical JSON, signature helpers, AG-UI-shaped UI events.       |
| `@gemmapod/shim`     | Browser runtime + Preact widget. Two IIFEs (full + runtime-only). Hand-curated `.d.ts`.     |
| `@gemmapod/browser`  | npm/CDN wrapper around the two shim IIFEs.                                                  |
| `@gemmapod/pack`     | `gemmapod` CLI: `init` / `keygen` / `doctor` / `build`.                                      |
| `@gemmapod/origin`   | Owner-side daemon. WebRTC + DARTC + signed tool registry + SQLite memory.                   |
| `@gemmapod/cloud`    | Reference signaling broker + pluggable `Registry` interface (Memory + SQLite ship).         |

### Docker images

| Image                                           | Notes                                                                 |
| ----------------------------------------------- | --------------------------------------------------------------------- |
| `ghcr.io/apprider/gemmapod-origin`              | Two-stage build; multi-arch (`linux/amd64`, `linux/arm64`).           |
| `ghcr.io/apprider/gemmapod-cloud`               | Two-stage build; multi-arch.                                          |

### Wire surface

- DARTC envelope: `v0.2`.
- UI event payload schema: `dartc.ui.event/0.1`.
- Signed manifest CBOR schema: `v=1`.

### What's not in v0.1

- Topic-level encryption (sender keys vs MLS — open question; see [DARTC spec §13](/docs/protocol/dartc-spec#13-open-questions)).
- WebSocket relay fallback (planned; envelope is transport-neutral above the layer).
- CBOR framing alternative to JSON.
- Capability-request flow with risk tiers (planned).
- CLI / server / worker environment adapters (planned).
- `crates.io` publish for `gemmapod-core`.
